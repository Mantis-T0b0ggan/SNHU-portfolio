01- Bookstore Database

The artifact that I chose first was the database that I designed and created in IT-330. The purpose of the database was to hold the inventory for a bookstore that had recently purchased a second store in a different location. A database was necessary for this problem because the inventory from both stores needed to be consolidated into one spot. That and because the owners wanted to begin keeping track of other inventory at the bookstores such as coffee, water and tea.

This artifact was chosen because it demonstrates that I have the skills, knowledge and abilities to create a database from scratch. There was problem solving that took place when trying to determine how each table could be related logically and what metadata to include and not include. The outcomes of the database can be seen in the screenshots that I submit for my final project in the class. In them, you can see all the tables that were created and where they are related and the queries on the database to display how the data has been stored and how it can be used.

In the process of creating the database I learned quite a lot. First how to use the SQL computer language and also how to effectively use UML design tools such as MS Visio. To design the database and the tables I had to use Visio a lot and got familiar with UML design tools. Along the way there were many challenges. One major challenge was creating enough tables to capture all the required inventory while also keeping clutter and repetition to a minimum. Creating the tables took many hours of planning, trial and error. Also, creating the correct queries for the SQL database was quite challenging. It took me a few attempts but as able to input all the data within the database with queries and then display the metadata within the database using SQL query.
